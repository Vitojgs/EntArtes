Sheet 5: RNF
ID Requisito Não Funcional Categoria Descrição
RNF01 Segurança de Credenciais Segurança O sistema deve armazenar palavras-passe utilizando algoritmos de hash seguros (ex: bcrypt), sem guardar palavras-passe em texto simples.
RNF02 Controlo de Acesso Segurança O sistema deve implementar controlo de acesso baseado em perfis de utilizador (Aluno, Encarregado, Professor, Direção).
RNF03 Tempo de Resposta Desempenho O sistema deve apresentar tempos de resposta adequados em condições normais de utilização e de acordo com os recursos disponíveis.
RNF04 Disponibilidade Confiabilidade O sistema deve estar disponível durante períodos de utilização previstos, podendo sofrer interrupções programadas para manutenção.
RNF05 Usabilidade Usabilidade O sistema deve apresentar uma interface clara e consistente, facilitando a navegação e utilização pelos utilizadores.
RNF06 Compatibilidade Portabilidade O sistema deve ser compatível com os principais navegadores modernos.
RNF07 Escalabilidade Desempenho O sistema deve permitir adaptação a um aumento de utilizadores através de melhorias na infraestrutura.
RNF08 Manutenibilidade Manutenção O sistema deve ser desenvolvido com arquitetura modular para facilitar manutenção, correção de erros e evolução.
RNF09 Integridade de Dados Confiabilidade O sistema deve garantir a integridade dos dados através de mecanismos de validação e restrições na base de dados.
RNF10 Backup e Recuperação Confiabilidade O sistema deve permitir a realização de backups periódicos e a recuperação de dados em caso de falha.
RNF11 Auditoria e Registo Segurança O sistema deve registar ações relevantes dos utilizadores para efeitos de monitorização e auditoria.
RNF12 Capacidade de Utilização Desempenho O sistema deve suportar múltiplos utilizadores simultâneos dentro da capacidade da infraestrutura utilizada.
